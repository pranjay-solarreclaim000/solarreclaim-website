# Purchasing Module

## 1. The Mandate
The Purchasing module is responsible for the strategic procurement of all software, hardware, and third-party services required to run the SolarReclaim operational engine. It owns vendor selection, contract negotiation, and budget adherence.
**Strictly Forbidden:** Signing multi-year contracts without CEO approval; committing to recurring spend over $500/mo without Finance verification.

## 2. Key Performance Indicators (KPIs)
- **Tool ROI:** Ratio of revenue generated vs. cost of the supporting tool stack.
- **Vendor Uptime/Reliability:** Tracking stability of critical infrastructure (dialers, CRMs).
- **Budget Variance:** Difference between projected software spend and actual monthly cost.

## 3. The Resource Map
- **Roles:**
    - Procurement Lead (Founder/Ops Manager)
    - Vendor Relationship Manager
- **Tools:**
    - Expense Management Tool (e.g., Ramp, Mercury)
    - Vendor Inventory Spreadsheet
    - Comparison Framework (Feature/Price Matrix)

## 4. The Interface
- **Inputs:**
    - Technology requirements (from IT/Product)
    - Approved procurement budget (from Finance)
- **Outputs:**
    - Active software licenses and access keys (to IT)
    - Monthly billing statements (to Finance)
